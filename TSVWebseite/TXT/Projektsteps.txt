Der angepasste Masterplan (Cloud Run + Neon)
Da wir nun PostgreSQL statt MySQL oder SQLite nutzen, ändern sich die Schritte leicht. Das macht das Ganze sogar einfacher, weil wir keine Datenbank-Datei mehr hin- und herschieben müssen!

Phase 1: Die externe Datenbank aufsetzen

Neon Account erstellen: Auf neon.tech registrieren und ein neues Projekt anlegen.

Tabelle anlegen: In der Neon-Konsole den SQL-Befehl ausführen, um deine posts-Tabelle zu erstellen (Achtung: PostgreSQL nutzt SERIAL statt AUTO_INCREMENT für IDs, das zeige ich dir dann).

Verbindungsdaten kopieren: Dir den Connection-String (URL, User, Passwort) von Neon kopieren.

Phase 2: Vorbereitung des Codes (Lokal)
4. api.php anpassen (Datenbank): Die PDO-Verbindung in deiner PHP-Datei von MySQL (mysql:host=...) auf PostgreSQL (pgsql:host=...) ändern. Die SQL-Befehle (SELECT, INSERT) bleiben fast identisch.
5. api.php anpassen (Bilder): Den Code so ändern, dass Bilder beim Upload in Google Cloud Storage gespeichert werden (da der Cloud Run Container ja "vergesslich" ist).
6. Dockerfile erstellen: Die Textdatei für Google Cloud schreiben (mit dem php:8.x-apache Image und der aktivierten PostgreSQL-Erweiterung pdo_pgsql).

Phase 3: Google Cloud Setup
7. GCP Projekt & API: Google Cloud Projekt erstellen und Cloud Run sowie Cloud Storage aktivieren.
8. Cloud Storage Bucket: Einen Speicherort (sg-nordangeln-images) für die Bild-Uploads anlegen.

Phase 4: Deployment (Der Live-Gang)
9. Image hochladen: Deinen Code über die Kommandozeile zu Google (Artifact Registry) pushen.
10. Cloud Run Service starten: Den Container starten. Wichtig hierbei: Du trägst die Neon-Datenbank-Zugangsdaten als "Umgebungsvariablen" (Environment Variables) in Cloud Run ein. So stehen sie nicht im Klartext in deinem Code – das ist hochprofessionell und sicher!
11. Domain verknüpfen: Deine Domain sg-nordangeln.de auf den Cloud Run Service leiten.
